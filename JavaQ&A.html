UNIT I ‚Äì OOPS Concepts

2 Marks Questions

1. Define Object-Oriented Programming (OOP).

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects". These objects can contain data in the form of fields (attributes or properties) and code in the form of procedures (methods). The primary goal is to model real-world entities to create more flexible and maintainable software.

2. What is encapsulation?

Encapsulation is the bundling of data (attributes) and the methods that operate on that data into a single unit, known as a class. It restricts direct access to some of an object's components, which is a key principle for protecting data from accidental modification (data hiding).

3. Define class and object.

A class is a blueprint or template for creating objects. It defines the properties (attributes) and behaviors (methods) that its objects will have. An object is a real-world instance of a class, with its own state and behavior.

4. What is polymorphism?

Polymorphism (from Greek, meaning "many shapes") is the ability of a method or object to take on different forms. In Java, it is achieved through method overloading (compile-time polymorphism) and method overriding (runtime polymorphism), allowing a single action to be performed in different ways.

5. List any two features of object orientation.

Two major features are:

  • Inheritance: A mechanism where a new class (subclass) inherits attributes and methods from an existing class (superclass).
  • Abstraction: The concept of hiding complex implementation details and showing only the essential features of the object to the user.

2. Explain the principles of object orientation with examples.

The four main principles of object orientation are:

  1. Encapsulation: Binding data and methods into a single unit (class) and hiding the data from the outside world. Example: A `BankAccount` class encapsulates the `balance` variable, making it private. The only way to change the balance is through public methods like `deposit()` and `withdraw()`.
  2. Abstraction: Hiding internal implementation details and showing only essential features. Example: When you use a TV remote, you press a button to change the channel without needing to know the complex electronics behind it.
  3. Inheritance: Allowing a class (child class) to inherit properties and methods from another class (parent class). Example: A `Dog` class and a `Cat` class can both inherit from an `Animal` class.
  4. Polymorphism: The ability to take more than one form. Example: An `Animal` class has a method `makeSound()`. The `Dog` object's `makeSound()` will print "Woof", while the `Cat` object's `makeSound()` will print "Meow".
3. Write short notes on inheritance and abstraction.

Inheritance: It is one of the core concepts of OOP that allows a class (child or subclass) to inherit the properties and behaviors (fields and methods) of another class (parent or superclass). This promotes code reusability and establishes an "is-a" relationship. The `extends` keyword is used to achieve inheritance in Java.

Abstraction: It is the concept of hiding the internal details and showing only the functionality. In Java, abstraction is achieved using abstract classes and interfaces. It focuses on what an object does instead of how it does it. For example, a `Shape` interface can declare a method `calculateArea()`, but the actual calculation is implemented differently by classes like `Circle` and `Square`.

4. Describe the features of an object ‚Äì attributes, state, and behavior.

An object in OOP is a self-contained entity that consists of three key features:

  • Attributes (or Properties): These are the data members of a class that represent the characteristics of an object. For a `Dog` object, attributes might include `color`, `breed`, and `age`.
  • State: The state of an object is represented by the current values of its attributes. For a specific `Dog` object, the state could be: `color` is "Brown", `breed` is "Labrador", `age` is 5.
  • Behavior: This defines what an object can do. It is represented by the methods of a class. For a `Dog` object, behaviors could be methods like `bark()`, `eat()`, and `sleep()`.
5. Explain modularity and message passing in OOP.

Modularity: Modularity is the property of a system that has been decomposed into a set of cohesive and loosely coupled modules. In OOP, classes and objects serve as these modules. A well-designed system is modular, meaning you can change the internal implementation of one object without affecting the other objects that interact with it.

Message Passing: Objects in an OOP system communicate with each other by sending and receiving messages. A message is essentially a request for an object to execute one of its methods. The receiving object then performs the action defined in that method. For example, a `Driver` object sends a "stop" message to a `Car` object by calling the `car.stop()` method.

1. Explain in detail the basic concepts and features of Object-Oriented Programming.

Object-Oriented Programming is a methodology to design a program using classes and objects. The main concepts are:

  • Class: A template or blueprint from which objects are created.
  • Object: An instance of a class, representing a real-world entity.
  • Encapsulation: The wrapping of data and methods into a single unit (a class). It implements data hiding, preventing direct access to data.
  • Inheritance: The mechanism by which one class acquires the properties and behaviors of a parent class, supporting code reusability.

2. Discuss various principles of object orientation with suitable examples in Java.

1. Encapsulation: Achieved by declaring class variables as private and providing public getter/setter methods.

public class Student { private String name; // Data hiding public String getName() { return name; } public void setName(String name) { this.name = name; } }

2. Inheritance: Using the `extends` keyword.

class Animal { void eat() { System.out.println("eating..."); } } class Dog extends Animal { void bark() { System.out.println("barking..."); } }

3. Polymorphism: Achieved via method overriding.

class Animal { void makeSound() { System.out.println("Animal sound"); } } class Cat extends Animal { @Override void makeSound() { System.out.println("Meow"); } }

4. Abstraction: Using `abstract` classes or `interfaces`.

abstract class Shape { abstract void draw(); } class Circle extends Shape { void draw() { System.out.println("drawing circle"); } }
3. Compare procedural programming and OOP with examples.

The core difference is in how data and operations are organized.

Procedural Programming (POP): Focuses on a sequence of instructions. Data and functions are separate entities.

Example in C (Procedural):

struct Rectangle { int length; int width; }; int getArea(struct Rectangle r) { return r.length * r.width; }

Object-Oriented Programming (OOP): Focuses on objects, where data and the functions that operate on that data are bundled together.

Example in Java (OOP):

class Rectangle { int length; int width; int getArea() { return length * width; } }

In the OOP example, the data and the operation are tightly coupled within the `Rectangle` class, which is more organized and secure.

UNIT II ‚Äì Introduction to Java

2 Marks Questions

1. What is JVM?

JVM (Java Virtual Machine) is an abstract machine that provides a runtime environment in which Java bytecode can be executed. It is a crucial component that enables Java's platform independence ("write once, run anywhere").

2. Define a bytecode.

Bytecode is the intermediate, platform-independent code generated by the Java compiler from the source code (`.java` file). It consists of a set of instructions for the JVM, which then interprets or compiles it into machine-specific code for execution.

3. What are primitive data types in Java?

Primitive data types are the most basic data types predefined by the Java language. They are not objects and store the actual values. There are eight: `byte`, `short`, `int`, `long`, `float`, `double`, `boolean`, and `char`.

4. What is a constructor?

A constructor in Java is a special method used to initialize objects. It is called when an object of a class is created. It must have the same name as the class and does not have a return type.

5. Define method overloading.

Method overloading is a feature that allows a class to have more than one method with the same name, as long as their parameter lists are different (i.e., different number, type, or order of parameters). It is a way to achieve compile-time polymorphism.

2. Write short notes on type casting and conversions in Java.

Type casting is when you assign a value of one primitive data type to another type.

1. Widening Conversion (Implicit): Happens automatically when converting a smaller data type to a larger type size. It is safe because there is no loss of information.

int myInt = 9; double myDouble = myInt; // Automatic casting: int to double

2. Narrowing Conversion (Explicit): Must be done manually when converting a larger data type to a smaller size type. There is a risk of losing information.

double myDouble = 9.78; int myInt = (int) myDouble; // Explicit casting: double to int. Result is 9.
3. Discuss static members and the this reference.

Static Members:

  • The `static` keyword is used for a variable or method that belongs to the class itself, rather than to any specific instance (object).
  • A static variable is shared among all instances of the class.
  • A static method can be called without creating an object of the class (`ClassName.staticMethod()`).

The `this` Reference:

  • `this` is a reference variable that refers to the current object.
  • It is used to resolve ambiguity between instance variables and parameters (`this.variable = variable`).
  • It can be used to invoke the current class's constructor (`this()`).
4. Explain dynamic method dispatch.

Dynamic Method Dispatch is the mechanism by which a call to an overridden method is resolved at runtime, rather than compile-time. It is how Java implements runtime polymorphism.

When an overridden method is called through a superclass reference, Java determines which version of that method to execute based on the type of the object being referred to at the time the call occurs.

class Animal { void move() { System.out.println("Animals can move"); } } class Dog extends Animal { void move() { System.out.println("Dogs can walk"); } } // In main: Animal a = new Dog(); // Superclass reference refers to subclass object a.move(); // Calls the method in the Dog class at runtime.
5. Describe different types of inheritance in Java.

Java supports the following types of inheritance:

  • Single Inheritance: A class inherits from only one superclass. (B extends A).
  • Multilevel Inheritance: A class inherits from another class, which in turn inherits from another class. (C extends B, and B extends A).
  • Hierarchical Inheritance: Multiple classes inherit from a single superclass. (B extends A, and C also extends A).

Note: Java does not support Multiple Inheritance with classes to avoid ambiguity. However, it can be achieved through interfaces.

1. Explain the main features of Java and its execution process.

Main Features of Java:

  • Platform Independent: Java's "Write Once, Run Anywhere" (WORA) is possible because code is compiled to bytecode that runs on any JVM.
  • Object-Oriented: Everything in Java is an object, promoting clean design and reusability.
  • Robust: Java has strong memory management, exception handling, and automatic garbage collection.
  • Secure: Provides a secure environment with a Security Manager and bytecode verifier.
  • Multithreaded: Built-in support for multithreading allows for concurrent execution.

Execution Process:

  1. Write Code: Source code is written in a `.java` file.
  2. Compile: The `javac` compiler translates it into bytecode (`.class` file).
  3. Load & Verify: The JVM's Classloader loads the bytecode, and the Bytecode Verifier checks it for security.
  4. Execute: The JVM's Interpreter or Just-In-Time (JIT) compiler converts the bytecode into native machine code for execution.
2. Discuss constructors and passing objects to methods with examples.

Constructors:

A constructor is a block of code similar to a method that's called when an instance of an object is created to initialize its state.

class Box { double width, height; // Parameterized Constructor Box(double w, double h) { width = w; height = h; } }

Passing Objects to Methods:

In Java, objects are passed to methods by reference. This means the method receives a reference to the original object, and any changes made to the object's state within the method will affect the original object.

class Test { int a; // Method that takes an object as a parameter void update(Test obj) { obj.a = obj.a * 2; } } // Usage in main: Test t = new Test(); t.a = 10; t.update(t); // Pass the object System.out.println(t.a); // Prints 20
3. Describe inheritance and method overriding in detail.

Inheritance:

Inheritance is a fundamental principle of OOP where a new class (subclass) inherits fields and methods from an existing class (superclass). It is achieved using the `extends` keyword. The main advantages are code reusability and establishing a clear hierarchy.

Method Overriding:

Method overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass. It is a way to achieve runtime polymorphism.

Rules for method overriding:

  • The method must have the same name, same parameter list, and same return type as in the parent class.
  • The access level cannot be more restrictive (e.g., cannot override a `public` method with a `private` one).
  • The `@Override` annotation is recommended for clarity.
class Vehicle { void run() { System.out.println("Vehicle is running"); } } class Bike extends Vehicle { @Override // Overriding the run method void run() { System.out.println("Bike is running safely"); } }

UNIT III ‚Äì Exceptions and Threads

2 Marks Questions

1. What is an exception in Java?

An exception is an unwanted or unexpected event that occurs during the execution of a program, disrupting the normal flow of instructions. In Java, an exception is an object that is "thrown" at runtime and can be "caught" and handled.

2. What is the use of the finally block?

The finally block is used to execute important code, such as closing files or database connections, regardless of whether an exception occurs in the `try` block or not. It is always executed after the `try-catch` block completes.

3. Define a thread.

A thread is the smallest unit of execution within a process. A Java application can have multiple threads running concurrently, allowing it to perform multiple tasks simultaneously (multithreading).

4. What is a daemon thread?

A daemon thread is a low-priority thread that runs in the background to provide services to user threads (e.g., the garbage collector). The JVM exits as soon as all user threads have completed, without waiting for daemon threads.

5. List any two thread states.

Two common thread states are:

  • NEW: The state of a thread when it has been created but not yet started.
  • RUNNABLE: The state of a thread that is ready to run and waiting for CPU time.

3. Discuss thread synchronization.

Thread synchronization is a mechanism that controls the access of multiple threads to any shared resource. It is necessary to prevent race conditions and data inconsistencies.

In Java, it can be achieved using:

  1. Synchronized Method: By declaring a method `synchronized`, only one thread can execute it at a time for a given object instance.
  2. Synchronized Block: By using a `synchronized(this)` block, you can lock a specific part of the code instead of the entire method, which can improve performance.
4. Explain thread life cycle.

The life cycle of a thread involves transitioning between several states:

  1. New: The thread is created but not yet started.
  2. Runnable: The thread is ready to execute after `start()` is called.
  3. Blocked/Waiting: The thread is temporarily inactive, waiting for a lock or an I/O operation.
  4. Timed Waiting: The thread is waiting for a specific amount of time (e.g., `Thread.sleep()`).
  5. Terminated (Dead): The thread has completed its execution.

1. Explain exception handling in Java with syntax and examples.

Exception handling is a mechanism to handle runtime errors to maintain the normal flow of the application. It is managed using `try`, `catch`, `finally`, `throw`, and `throws`.

Syntax:

try { // Code that may throw an exception } catch (ExceptionType1 e1) { // Code to handle exception } finally { // Code that will always be executed }

Example:

public class ExceptionHandlingExample { public static void main(String args[]) { try { int data = 100 / 0; // Throws ArithmeticException } catch (ArithmeticException e) { System.out.println("Cannot divide by zero."); } finally { System.out.println("Finally block is executed."); } } }
2. Discuss thread creation methods and synchronization in detail.

Thread Creation Methods:

1. Extending the `Thread` Class: Create a new class that extends `Thread` and override its `run()` method. Limitation: This class cannot extend any other class.

2. Implementing the `Runnable` Interface: Create a class that implements `Runnable` and implement the `run()` method. This is preferred as it allows the class to extend another class.

Synchronization:

Synchronization is crucial for controlling access to shared resources. It is achieved using the `synchronized` keyword.

  • Synchronized Method: When a method is declared `synchronized`, the thread acquires a lock on the object before executing the method.
  • Synchronized Block: Provides more granular control by locking only a part of the method's code, reducing the scope of the lock.
3. Explain the concept of multithreading in Java with suitable examples.

Multithreading is a Java feature that allows concurrent execution of two or more parts of a program for maximum utilization of the CPU. Each part of such a program is called a thread.

Advantages: Improved Performance, Responsiveness, and efficient Resource Sharing.

Example:

class MultiTask implements Runnable { public void run() { /* task logic */ } } public class MultiThreadExample { public static void main(String[] args) { Thread thread1 = new Thread(new MultiTask()); Thread thread2 = new Thread(new MultiTask()); thread1.start(); thread2.start(); } }

In this example, the two tasks will execute concurrently, with their outputs interleaved in the console.

UNIT IV ‚Äì JDBC and Applets

2 Marks Questions

1. What is JDBC?

JDBC (Java Database Connectivity) is a standard Java API for connecting and executing queries with a database. It provides classes and interfaces to communicate with various relational databases.

2. What is a ResultSet in JDBC?

A ResultSet is a Java object that represents a table of data generated by executing a database query (like a `SELECT` statement). It maintains a cursor pointing to its current row of data.

3. Define an applet.

An applet is a special type of small Java program that is designed to be embedded within another application, typically a web page, and executed in a client's web browser. Applets are now considered deprecated technology.

4. What is an SQL query?

An SQL (Structured Query Language) query is a command used to interact with a relational database. It can be used to perform tasks such as retrieving data (`SELECT`), updating data (`UPDATE`), and deleting data (`DELETE`).

5. What is executeQuery() used for?

The `executeQuery()` method in JDBC is used specifically for executing SQL statements that retrieve data, primarily `SELECT` queries. It returns the results in a `ResultSet` object.

5 Marks Questions

1. Explain steps to connect a Java program with a database using JDBC.

There are 5 main steps to connect to a database using JDBC:

  1. Load and Register the Driver: Load the database driver class into memory using `Class.forName("com.mysql.cj.jdbc.Driver");`.
  2. Establish a Connection: Use the `DriverManager.getConnection()` method to create a `Connection` object. This requires the database URL, username, and password.
  3. Create a Statement: Use the `connection.createStatement()` method to create a `Statement` or `PreparedStatement` object for sending SQL queries to the database.
  4. Execute the Query: Use a method like `executeQuery()` (for SELECT) or `executeUpdate()` (for INSERT, UPDATE, DELETE) on the statement object.
  5. Close the Connection: Close the `ResultSet`, `Statement`, and `Connection` objects in a `finally` block to release database resources.
2. Write short notes on JDBC driver types.

There are four types of JDBC drivers:

  • Type 1 (JDBC-ODBC Bridge Driver): Translates JDBC calls into ODBC calls. Now deprecated.
  • Type 2 (Native-API Driver): Uses a vendor-specific native library on the client side. Not fully written in Java.
  • Type 3 (Network-Protocol Driver): Uses middleware to communicate with the database. Platform-independent.
  • Type 4 (Thin Driver): A pure Java driver that communicates directly with the database's network protocol. This is the most common type used today.
3. Describe applet life cycle.

The applet life cycle is managed by the web browser and consists of five main methods:

  1. `init()`: Called once when the applet is first loaded. Used for one-time initialization.
  2. `start()`: Called after `init()` and every time the user returns to the page containing the applet. Used to start animations or threads.
  3. `stop()`: Called when the user leaves the page containing the applet. Used to stop animations or threads.
  4. `destroy()`: Called once when the applet is unloaded from memory (e.g., when the browser is closed). Used for final cleanup.
  5. `paint()`: Called whenever the applet's display needs to be redrawn.

UNIT V ‚Äì Events and GUI Applications

2 Marks Questions

1. What is event handling?

Event handling is the mechanism for responding to events, which are actions initiated by the user, such as clicking a button, moving the mouse, or typing a key. It involves an event source (the component that generates the event) and an event listener (an object that is notified when the event occurs).

2. What is AWT?

AWT (Abstract Window Toolkit) is Java's original platform-dependent API for creating Graphical User Interfaces (GUIs). It provides a set of heavyweight components like buttons and labels that rely on the underlying native operating system's GUI toolkit.

3. What is an adapter class?

An adapter class provides an empty implementation of all methods in an event listener interface. They are convenience classes that you can extend if you are only interested in handling a subset of the events in an interface, saving you from having to provide empty implementations for all other methods.

4. Define layout manager.

A layout manager is an object that controls the positioning and sizing of components within a container. It automatically arranges components, making GUIs more portable. Examples include `FlowLayout`, `BorderLayout`, and `GridLayout`.

5. What is a component?

A component is the base class for all AWT GUI objects. It is an abstract object that can be displayed on the screen and can interact with the user. Examples include buttons, text fields, labels, and checkboxes.

5 Marks Questions

1. Explain the delegation event model in Java.

The Delegation Event Model defines a standard mechanism for generating and handling events. It has three key components:

  1. Event Source: The object that generates an event (e.g., a `Button`).
  2. Event Listener: An object that is notified when an event occurs. It must implement a specific listener interface (e.g., `ActionListener`).
  3. Event Object: An object that encapsulates information about the event that occurred (e.g., an `ActionEvent`).

The process works as follows: The listener registers itself with the source. When the source generates an event, it notifies all registered listeners by calling a specific method on them, passing the event object as an argument.

2. Describe various AWT components and containers.

AWT Components (Controls): These are the basic GUI elements that the user interacts with.

  • `Button`: A push button that triggers an action.
  • `Label`: A display area for a single line of read-only text.
  • `TextField`: A single-line text entry area.
  • `Checkbox`: A two-state (on/off) selection box.

AWT Containers: These are components that can hold other components.

  • `Frame`: A top-level window with a title and a border.
  • `Panel`: A generic container used to group components inside a `Frame`.
  • `Dialog`: A pop-up window that takes input from the user.
3. Explain event classes and event listener interfaces.

Event Classes: Each type of event is represented by a class. These classes are in the `java.awt.event` package.

  • `ActionEvent`: Generated by a button click, menu selection, etc.
  • `MouseEvent`: Generated by mouse actions like clicks, movement, etc.
  • `KeyEvent`: Generated by keyboard key presses.

Event Listener Interfaces: For each event class, there is a corresponding listener interface that defines the methods to be called when the event occurs.

  • `ActionListener`: For `ActionEvent`. Has one method: `actionPerformed()`.
  • `MouseListener`: For `MouseEvent`. Has methods like `mousePressed()`, `mouseClicked()`, etc.
  • `KeyListener`: For `KeyEvent`. Has methods like `keyPressed()`, `keyReleased()`.
4. Write short notes on layout managers.

Layout managers automatically arrange components within a container.

  • `FlowLayout`: The default for `Panel`. Arranges components in a left-to-right flow.
  • `BorderLayout`: The default for `Frame`. Divides the container into five regions: NORTH, SOUTH, EAST, WEST, and CENTER.
  • `GridLayout`: Arranges components in a rectangular grid of rows and columns.
  • `GridBagLayout`: The most flexible and complex layout manager. It aligns components vertically and horizontally without requiring them to be of the same size.
5. Describe how to handle button click events.

Handling a button click event involves these steps:

  1. Create a Component: Create an instance of the `Button` class (the Event Source).
  2. Implement the Listener: Create a class that implements the `ActionListener` interface. This class must provide an implementation for the `actionPerformed(ActionEvent e)` method (the Event Listener).
  3. Register the Listener: Add the listener object to the button using the `button.addActionListener(listener)` method.

When the button is clicked, it creates an `ActionEvent` and calls the `actionPerformed()` method on all its registered listeners.

Button b = new Button("Click Me"); b.addActionListener(new ActionListener() { public void actionPerformed(ActionEvent e) { System.out.println("Button was clicked!"); } });

1. Explain the AWT hierarchy and its components in detail.

The AWT class hierarchy is rooted in the `Object` class.

  • `Component`: An abstract base class for all GUI elements.
  • `Container`: A subclass of `Component` that can hold other components.

The hierarchy branches from `Container`:

  • `Window`: A top-level window.
    • `Frame`: A fully functional window with a title and border. The main container for applications.
    • `Dialog`: A temporary pop-up window.
  • `Panel`: A generic, borderless container used to group components.
    • `Applet`: A subclass of `Panel` designed for web browsers.

Direct subclasses of `Component` are the controls like `Button`, `Label`, `TextField`, etc. This hierarchy allows for a structured way to build complex user interfaces.

2. Discuss event handling mechanism in Java with suitable examples.

Java's event handling mechanism is based on the Delegation Event Model, which consists of three parts:

  1. Event Source: A component that generates an event (e.g., `Button`).
  2. Event Listener: An object that listens for events. It must implement a listener interface (e.g., `ActionListener`).
  3. Event Object: An object created by the source to encapsulate event details (e.g., `ActionEvent`).

Mechanism:

  1. The listener object registers itself with a source using an `addListener()` method.
  2. When the user interacts with the source, the source creates an event object.
  3. The source then calls the appropriate method on all its registered listeners, passing the event object.

Example (Handling a Mouse Click):

import java.awt.*; import java.awt.event.*; public class MouseEventExample extends Frame implements MouseListener { Label l; MouseEventExample() { l = new Label(); l.setBounds(20, 50, 100, 20); add(l); addMouseListener(this); // Registering listener setSize(300,300); setLayout(null); setVisible(true); } public void mouseClicked(MouseEvent e) { l.setText("Mouse Clicked"); } // Other MouseListener methods must also be implemented... public void mousePressed(MouseEvent e) {} public void mouseReleased(MouseEvent e) {} public void mouseEntered(MouseEvent e) {} public void mouseExited(MouseEvent e) {} }
3. Explain the creation of a simple GUI using AWT controls.

Creating a simple GUI involves these steps:

  1. Choose a Container: Start with a top-level container, typically a `Frame`.
  2. Set Layout: Choose and set a layout manager (e.g., `frame.setLayout(new FlowLayout());`).
  3. Create Components: Create instances of AWT controls like `Label`, `TextField`, and `Button`.
  4. Add Components to Container: Use the `add()` method to place the components into the container.
  5. Handle Events: Implement and register event listeners for interactive components.
  6. Set Frame Properties: Set the size, visibility, and closing operation of the frame.

Example: A Simple Login Form

import java.awt.*; import java.awt.event.*; public class SimpleGui extends Frame { SimpleGui() { setLayout(new FlowLayout()); add(new Label("Username:")); add(new TextField(20)); add(new Label("Password:")); TextField passText = new TextField(20); passText.setEchoChar('*'); add(passText); add(new Button("Login")); addWindowListener(new WindowAdapter() { public void windowClosing(WindowEvent we) { System.exit(0); } }); setTitle("Login Form"); setSize(300, 150); setVisible(true); } }
