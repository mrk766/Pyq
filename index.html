Unit I – OOPS Concepts

1. Define Object-Oriented Programming (OOP).
OOP is a programming paradigm based on the concept of objects that contain data and methods to operate on that data.
2. List the main principles of OOP.
Abstraction, Encapsulation, Inheritance, and Polymorphism.
3. What is a class in Java?
A class is a blueprint or template from which objects are created. It defines the properties and behaviors that objects of that class will have.
4. What is an object?
An object is an instance of a class that has state (attributes) and behavior (methods).
5. Explain encapsulation.
Encapsulation is the mechanism of wrapping data (variables) and code (methods) together as a single unit and restricting direct access to some of the object's components.
6. What is inheritance?
Inheritance is a mechanism where a new class (child/subclass) derives properties and behaviors from an existing class (parent/superclass).
7. Explain polymorphism.
Polymorphism allows objects of different classes to be treated as objects of a common superclass and enables methods to do different things based on the object that invokes them.
8. What is abstraction?
Abstraction is the process of hiding implementation details and showing only essential features of an object.
9. Differentiate between method overloading and overriding.
Overloading: Same method name, different parameters (compile-time polymorphism). Overriding: Same method signature in subclass (runtime polymorphism).
10. What are access modifiers in Java?
Access modifiers define the visibility of classes, methods, and variables. Types: private, default, protected, public.

Unit II – Introduction to Java

1. What is bytecode in Java?
Bytecode is an intermediate code generated by the Java compiler that runs on the JVM.
2. What is JVM?
JVM (Java Virtual Machine) is an abstract machine that provides runtime environment to execute Java bytecode.
3. What is JRE?
JRE (Java Runtime Environment) contains JVM and class libraries required to run Java applications.
4. What is JDK?
JDK (Java Development Kit) contains JRE plus development tools like compiler, debugger, etc.
5. List features of Java.
Platform independent, object-oriented, secure, robust, portable, multithreaded, etc.
6. What are data types in Java?
Primitive: byte, short, int, long, float, double, char, boolean. Non-primitive: Arrays, Strings, Classes, Interfaces.
7. What are variables in Java?
Variables are containers for storing data values. Types: local, instance, static.
8. What are operators in Java?
Operators are symbols that perform operations on operands. Types: arithmetic, relational, logical, assignment, etc.
9. Explain control statements.
Control statements control the flow of execution. Types: decision making (if, switch), looping (for, while, do-while), branching (break, continue, return).
10. What are arrays in Java?
Arrays are objects that store multiple variables of the same type. They have fixed size once created.

Unit III – Exceptions and Multithreading

1. What is an exception?
An exception is an event that disrupts the normal flow of program execution.
2. Difference between checked and unchecked exceptions.
Checked: Compile-time exceptions (IOException, SQLException). Unchecked: Runtime exceptions (NullPointerException, ArrayIndexOutOfBounds).
3. What is exception handling?
Exception handling is the process of responding to exceptions using try, catch, finally, throw, and throws keywords.
4. What is multithreading?
Multithreading is a programming concept where multiple threads run concurrently within a single program.
5. How to create threads in Java?
By extending Thread class or implementing Runnable interface.
6. What is synchronization?
Synchronization is the capability to control the access of multiple threads to shared resources.
7. What is thread lifecycle?
New, Runnable, Running, Blocked/Waiting, Terminated.
8. Difference between wait() and sleep().
wait() releases lock, sleep() doesn't. wait() is from Object, sleep() is from Thread.
9. What are collection frameworks?
Collection framework provides architecture to store and manipulate group of objects.
10. List common collection interfaces.
List, Set, Map, Queue, Deque.
11. What is ArrayList?
ArrayList is a resizable array implementation of the List interface.
12. What is HashMap?
HashMap is a hash table based implementation of the Map interface.
13. Difference between ArrayList and LinkedList.
ArrayList: Random access fast, insertion/deletion slow. LinkedList: Insertion/deletion fast, random access slow.
14. What is HashSet?
HashSet is an implementation of Set interface that doesn't allow duplicate elements.
15. What is Iterator?
Iterator is an interface used to iterate through collections.

Unit IV – JDBC and Database Connectivity

1. What is JDBC?
JDBC (Java Database Connectivity) is an API that enables Java programs to execute SQL statements.
2. What are JDBC drivers?
JDBC drivers implement the JDBC API for specific databases. Types: Type 1 to Type 4.
3. Steps to connect to database using JDBC.
1. Load driver, 2. Create connection, 3. Create statement, 4. Execute query, 5. Process results, 6. Close connection.
4. Difference between Statement and PreparedStatement.
PreparedStatement is precompiled and more efficient for repeated executions. It prevents SQL injection.
5. What is connection pooling?
Connection pooling maintains a cache of database connections that can be reused.
6. What is transaction management?
Transaction management ensures database operations are completed as a single unit (all or nothing).
7. What is ResultSet?
ResultSet object maintains a cursor pointing to its current row of data and provides methods to retrieve column values.
8. What is batch processing in JDBC?
Batch processing allows grouping related SQL statements into a batch and executing them together.
9. What are JDBC exceptions?
SQLException is the base exception class for JDBC operations.
10. What is metadata in JDBC?
Metadata provides information about the database (DatabaseMetaData) and result set (ResultSetMetaData).
11. How to handle transactions in JDBC?
Using setAutoCommit(false), commit(), and rollback() methods.
12. What is CallableStatement?
CallableStatement is used to execute stored procedures in the database.
13. What is DatabaseMetaData?
DatabaseMetaData provides information about the database like supported features, version, etc.
14. What is ResultSetMetaData?
ResultSetMetaData provides information about the columns in a ResultSet.
15. What are the different types of JDBC statements?
Statement, PreparedStatement, and CallableStatement.

Unit V – GUI Programming with AWT and Swing

1. What is AWT?
AWT (Abstract Window Toolkit) is Java's original platform-dependent GUI toolkit.
2. What is Swing?
Swing is a lightweight, platform-independent GUI toolkit built on top of AWT.
3. Difference between AWT and Swing.
AWT is heavyweight (uses OS components), Swing is lightweight (pure Java). Swing has more components and better look-and-feel.
4. What are layout managers?
Layout managers automatically arrange components in containers. Examples: FlowLayout, BorderLayout, GridLayout.
5. What is event handling?
Event handling is the mechanism to handle user interactions with GUI components.
6. What are event listeners?
Event listeners are interfaces that contain methods to handle specific types of events.
7. List common Swing components.
JFrame, JButton, JLabel, JTextField, JTextArea, JComboBox, JCheckBox, JRadioButton, JMenu, etc.
8. What is MVC architecture?
MVC (Model-View-Controller) separates application into three components: Model (data), View (UI), Controller (logic).
9. What is applet?
Applet is a Java program that runs in a web browser. (Note: Deprecated in modern Java)
10. What is JFrame?
JFrame is a top-level container that represents the main window of a Swing application.
11. What is JPanel?
JPanel is a generic lightweight container used to group components.
12. What are the different event types in AWT/Swing?
ActionEvent, MouseEvent, KeyEvent, WindowEvent, etc.
13. What is adapter class?
Adapter classes provide empty implementations of listener interfaces to reduce code.
14. What is the difference between paint() and repaint()?
paint() is called automatically, repaint() requests a call to paint().
15. What is Graphics class?
Graphics class provides methods for drawing shapes, text, and images on components.
