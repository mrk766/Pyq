Sequential Logic Circuits - Q&A Notes

1. Sequential Circuits

2 Marks: Definition

A sequential circuit is a digital circuit whose output depends on both present inputs and past history (stored in memory elements).

5 Marks: Explanation

Sequential circuits contain memory elements (like flip-flops) and combinational logic. Output changes according to input and stored states. Examples: flip-flops, registers, counters.

10 Marks: Detailed Answer

Sequential circuits are classified into synchronous (clock-controlled) and asynchronous (immediate response) circuits. They are used for data storage, sequence generation, and counters. They are fundamental for designing memory and processor components.

2. Latch (SR / D)

2 Marks: Definition

A latch is a level-triggered 1-bit memory device. It stores data when enable signal is active.

5 Marks: Working (SR Latch)

SR Latch has inputs S (Set) and R (Reset). Outputs Q and Q̅ store 1 bit.

10 Marks: D Latch Explanation

D Latch has single input D and follows input when Enable = 1. Used in registers, it simplifies circuit design and prevents invalid state.

3. Flip-Flops

2 Marks: Definition

Flip-flops are bistable circuits that store 1 bit. Output changes only on clock trigger.

5 Marks: SR / JK / D Flip-Flop

SR Flip-Flop: Set/Reset operation with Q and Q̅ outputs.

JK Flip-Flop: Improved SR; toggles when J=K=1.

D Flip-Flop: Output follows D input at clock edge.

10 Marks: Master-Slave & T Flip-Flop

Master-Slave JK prevents race by edge triggering. T Flip-Flop toggles output each clock if T=1. Used in counters and memory storage.

A shift register is a series of flip-flops used to store and shift data on clock pulses.

5. Counters

2 Marks: Ring & Johnson Definition

Ring Counter: circulates a single 1 through N flip-flops (N states).
Johnson Counter: inverted output of last FF fed to first (2N states).

5 Marks: Ring Counter Example

4-bit Ring Counter states: 1000 → 0100 → 0010 → 0001 → repeat.

10 Marks: Johnson Counter Example

4-bit Johnson Counter states: 0000 → 0001 → 0011 → 0111 → 1111 → 1110 → 1100 → 1000 → repeat.

6. Instruction Format

2 Marks: Definition

Instruction format defines how an instruction is divided into opcode and operands.

5 Marks: Fields
  • Opcode: Operation code (ADD, SUB, LOAD)
  • Address: Memory/register location of operands
  • Mode: Addressing mode (immediate, direct, indirect)
  • Register: Register involved
10 Marks: Example

ADD R1, X → Opcode: ADD, Operand1: R1, Operand2: Memory X. Instruction controls CPU operation and data access.
