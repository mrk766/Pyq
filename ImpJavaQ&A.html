Definition for Exam

The thread lifecycle in Java represents the sequence of states a thread transitions through from its creation to its termination. These states are New, Runnable, Running, Blocked/Waiting, and Terminated. The transition between states is managed by the JVM's thread scheduler in response to method calls like start(), sleep(), and wait().

Definition for Exam

The applet lifecycle consists of a series of methods invoked by the browser to manage its execution. The primary methods are init() for one-time initialization, start() to begin or resume execution when the applet becomes visible, stop() to pause execution when the applet is hidden, and destroy() to perform final cleanup before the applet is unloaded from memory.

Definition for Exam

The AWT (Abstract Window Toolkit) hierarchy is the class inheritance structure for all of Java's core GUI components. It is rooted in the Component class, the base for all visible elements. The Container class extends Component and can hold other components, forming a tree-like structure displayed within a top-level window like a Frame.

Definition for Exam

Exception handling is a mechanism in Java to manage runtime errors and maintain the normal flow of the application. It uses five keywords: try to enclose code that may produce an exception, catch to handle it, finally to execute cleanup code, throw to manually raise an exception, and throws to declare that a method may produce an exception.

Easy Explanation

Analogy: A safety net for your code.

  • try: The risky code goes here.
  • catch: If an error happens, this block runs.
  • finally: This code runs NO MATTER WHAT.

Definition for Exam

Inheritance is a core concept of Object-Oriented Programming where a new class (subclass) derives the properties and behaviors (fields and methods) from an existing class (superclass). It is implemented using the extends keyword, promotes code reusability, and establishes an "is-a" relationship between the parent and child classes.

Easy Explanation

Analogy: A child inheriting traits from a parent. A `Car` is a type of `Vehicle`.

  • Promotes code reuse.
  • Uses the `extends` keyword.
  • Creates an "is-a" relationship (e.g., A `Dog` is an `Animal`).

Definition for Exam

An abstract class, declared with the abstract keyword, is a class that cannot be instantiated. It serves as a blueprint for its subclasses, and can contain both abstract methods (without a body) and concrete methods. Any non-abstract subclass that extends an abstract class must provide an implementation for all inherited abstract methods.

Easy Explanation

Analogy: A blueprint for a building. You can't live in the blueprint, but you must follow its rules (implement its abstract methods) when you build a real house from it.

  • Cannot be instantiated (can't create an object from it).
  • Can have both regular and abstract (body-less) methods.
  • Forces subclasses to follow a specific structure.
